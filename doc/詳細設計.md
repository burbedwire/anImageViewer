# 詳細設計書

## 1. 概要

### 1.1 製品名
ImageViewer

### 1.2 製品概要
Windows上で動作する画像ビューワーアプリケーション。画像の表示、クリップ、解像度変換、透過色指定、一括変換の機能を提供する。

### 1.3 開発環境

| 項目 | 内容 |
|-----|------|
| OS | Windows 10/11 |
| 言語 | Python 3.14 |
| 仮想環境 | venv |
| IDE | Visual Studio Code |

## 2. 技術構成

### 2.1 使用ライブラリ

| ライブラリ | バージョン | 用途 |
|-----------|-----------|------|
| customtkinter | 最新版 | モダンなGUIコンポーネント（テーマ対応） |
| Pillow | 最新版 | 画像の読み込み、加工、保存 |
| tkinterdnd2 | 最新版 | ドラッグアンドドロップ機能 |
| pytest | 最新版 | テストフレームワーク |
| pyinstaller | 最新版 | 単一EXEへのパッケージング |

### 2.2 アーキテクチャ

アプリケーションは以下の層で構成される：

```
┌─────────────────────────────────────────┐
│           プレゼンテーション層             │
│  (ImageViewerApp, Dialogクラス群)         │
├─────────────────────────────────────────┤
│           ビジネスロジック層               │
│  (画像処理、マスク計算、サイズ計算)          │
├─────────────────────────────────────────┤
│             データアクセス層               │
│  (Pillowによる画像I/O)                   │
└─────────────────────────────────────────┘
```

### 2.3 設計思想

- **シンプルさ**: 単一ファイルに機能をまとめることで、保守性を高める
- **非破壊編集**: 元の画像は変更せず、新規ファイルとして保存する
- **ユーザーフレンドリー**: ドラッグアンドドロップ、キーボードショートカットによる直感的な操作
- **レスポンシブ表示**: ウィンドウサイズに合わせて画像を縮小表示

### 3.2 クラス構成

#### 3.2.1 ImageViewerApp (app.py)

画像ビューワーのメインウィンドウクラス。

**継承**: `ctk.CTk`

**主要属性**:

- `_image_list: List[Path]` - 現在表示中のフォルダの画像リスト

- `_photo_image: Optional[ImageTk.PhotoImage]` - キャンバス表示用のPhotoImage
- `_canvas_image_item: Optional[int]` - キャンバス上の画像アイテムID

**主要メソッド**:

| メソッド | 説明 |
|---------|------|
| `open_image(file_path: str) -> bool` | 画像ファイルを開く |
| `_display_image(img: Image.Image) -> None` | キャンバスに画像を表示 |
| `_prev_image(event: tk.Event) -> None` | 前の画像を表示 |
| `_next_image(event: tk.Event) -> None` | 次の画像を表示 |
| `_first_image(event: tk.Event) -> None` | 1枚目の画像を表示 |
| `_last_image(event: tk.Event) -> None` | 最後の画像を表示 |
| `_open_file() -> None` | ファイルオープンダイアログ表示 |
| `_close_file() -> None` | 現在開いているファイルを閉じる |
| `_clip_save() -> None` | クリップして保存ダイアログ表示 |
| `_start_clip() -> None` | クリップモード開始 |
| `_set_transparency() -> None` | 透過色指定ダイアログ表示 |
| `_resize_image() -> None` | 解像度変換ダイアログ表示 |
| `_batch_convert() -> None` | 一括変換ダイアログ表示 |

## 5. API

### 5.1 キーボードショートカット

| ショートカット | 機能 |
|--------------|------|
| Ctrl+O | 画像を開く |
| Ctrl+W | 閉じる |
| Ctrl+C | クリップ |
| Ctrl+Shift+C | クリップして保存 |
| Ctrl+T | 透過色指定 |
| Ctrl+R | 解像度変換 |
| Ctrl+Shift+R | 一括変換 |
| ← | 前の画像 |
| → | 次の画像 |
| ↑ | 1枚目の画像 |
| ↓ | 最後の画像 |

### 5.2 透過マスク計算アルゴリズム

透過マスクは以下のアルゴリズムで計算される：

1. ユーザーが画像をクリックして背景色（R, G, B）を選択
2. 各画素について、背景色とのユークリッド距離を計算
   ```
   distance = sqrt((r - r_bg)² + (g - g_bg)² + (b - b_bg)²)
   ```
3. 距離が許容範囲（tolerance × 4.42）以内の画素を透過（マスク=0）とする
4. 許容範囲外の画素は非透過（マスク=255）とする

### 5.3 リサイズ計算

リサイズは以下の3つのモードをサポート：

| モード | 計算式 |
|-------|--------|
| 比率指定 | `new_w = orig_w × ratio / 100`, `new_h = orig_h × ratio / 100` |
| 幅指定 | `ratio = new_w / orig_w`, `new_h = orig_h × ratio` |
| 高さ指定 | `ratio = new_h / orig_h`, `new_w = orig_w × ratio` |

すべてのモードで `max(1, 計算値)` により1ピクセル以上を確保する。

## 8. エラー処理

- 画像ファイルが見つからない場合: ステータスバーに表示
- サポートされていない形式の場合: エラーメッセージ表示
- 保存失敗時: messageboxでエラー内容を表示
- ダイアログの早期閉じ: `tk.TclError` をキャッチして無視

## 9. ロギング

loggingモジュールを使用して、以下のレベルでログを出力：

| レベル | 使用箇所 |
|-------|---------|
| INFO | 画像の読み込み、保存、変換完了 |
| WARNING | DND初期化失敗、未開画状態での機能呼び出し |
| ERROR | 画像読み込み失敗、保存失敗、変換失敗 |
